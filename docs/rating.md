# Рейтинг формуласы

Adal Jumys рейтингі толық ашық: осы құжат формуланың өзін, барлық константаны және неге дәл осылай таңдалғанын түсіндіреді. Кодтағы іске асыруы: `backend/app/services/rating.py`. Формула өзгерсе, бұл құжат та бірге өзгереді.

Ешқандай ақылы қызмет рейтингке әсер ете алмайды. Бұл платформаның негізгі ұстанымы.

# Екі ось, екі түрлі көрсеткіш

Компанияның екі бөлек көрсеткіші бар және олар әдейі әртүрлі форматта:

1. **Жұмыс беруші рейтингі** (сандық, 1-10): қызметкер отзывтарынан есептеледі. Төмендегі формула осыған қатысты.
2. **Жалдаушы көрсеткіші** (сан ЕМЕС): кандидат шағымдарының ашық статистикасы (категория бойынша сандар) және паттерн-бейджтер. Санға айналдырмау себебі: шағымды тек ренжіген кандидат жазады (іріктеу қателігі), ал "қанағаттанған кандидат" деген ағын жоқ. Мұндай дерек көзінен шығарылған сан статистикалық тұрғыда алдамшы болар еді.

# Жұмыс беруші рейтингінің формуласы

Салмақты Байес орташасы:

```
рейтинг = (Σ (w_i × s_i) + m × C) / (Σ w_i + m)
```

мұндағы:

- `s_i` - әр отзывтың жалпы бағасы (1-10)
- `w_i` - әр отзывтың салмағы (төменде)
- `m = 5` - сенімділік табалдырығы
- `C` - платформадағы барлық жарияланған отзывтың қарапайым орташасы (отзыв мүлдем аз кезде C = 5.5, шкала ортасы)

# Отзыв салмағы

Әр отзывтың салмағы екі көбейткіштен тұрады:

```
w = базалық_салмақ × 0.5 ^ (жасы_күнмен / 912.5)
```

1. **Базалық салмақ**: верифицирленген отзыв (жұмыс фактісі модераторға дәлелденген) = 1.0, верифицирленбеген = 0.4. Верифицирленбеген дауыс та естіледі, бірақ рейтингті негізінен расталған тәжірибе қозғайды. Бұл фейк-отзыв шабуылының әсерін де күрт азайтады.
2. **Уақыт-decay**: салмақ 2.5 жылда (912.5 күн) екі есе азаяды. Ескі кейс компанияны мәңгі жазаламайды: басшылық ауысып, жағдай түзелуі мүмкін. Жаңа отзывтар келсе, рейтинг соларға қарай жылжиды.

# Неге Байес орташасы

Қарапайым орташаның осал жері: жалғыз 10/10 отзывы бар компания жүз отзывты, орташасы 8.0 компаниядан "жоғары" тұрып қалады. Байес орташасы аз дерек кезінде бағаны платформа орташасына (C) тартып ұстайды: сенімді ауытқу үшін жеткілікті салмақ жиналуы керек.

`m = 5` деген мағына: рейтинг платформа орташасынан айтарлықтай ауытқуы үшін кемінде ~5 толық салмақты (верифицирленген, жаңа) отзыв керек.

# Мысалдар

Платформа орташасы C = 5.0 делік.

**Мысал 1: жалғыз мақтау-отзыв.** Компанияда бір ғана жаңа, верифицирленбеген 10/10 отзыв бар (w = 0.4):

```
рейтинг = (0.4 × 10 + 5 × 5.0) / (0.4 + 5) = 29 / 5.4 = 5.4
```

10 емес, 5.4: жалғыз дауыс рейтингті ұшыра алмайды.

**Мысал 2: расталған нашар тәжірибе.** Үш жаңа верифицирленген отзыв: 2, 3, 2 (әрқайсысы w = 1.0):

```
рейтинг = (2 + 3 + 2 + 5 × 5.0) / (3 + 5) = 32 / 8 = 4.0
```

**Мысал 3: уақыт әсері.** Сол компанияға 5 жылдан кейін (ескі салмақтар 4 есе азайған: w = 0.25) төрт жаңа верифицирленген 8/8/8/8 отзыв келді:

```
рейтинг = (0.25×(2+3+2) + 1.0×(8×4) + 5×5.0) / (0.75 + 4 + 5) = (1.75 + 32 + 25) / 9.75 = 6.0
```

Компания түзелгені рейтингте көрінеді, бірақ тарих те жоғалмайды.

# Қандай отзывтар есепке кіреді

Тек көрінетіндер: жарияланған, модератор жасырмаған, авторы өшірмеген. Жасырылған отзыв рейтингтен де шығады (себебі жария аудит-логта жазулы).

# Паттерн-бейджтер

Бейджтер де ашық ережемен, автоматты есептеледі. Тәуелсіздік деген: бөлек аккаунттар (1 автор = 1 дауыс), тек жарияланған әрі жасырылмаған контент.

| Бейдж | Шарты |
| --- | --- |
| Жарияланған жалақы расталмайды | 5+ тәуелсіз автордан жалақы-алдау шағымы |
| Тілдік кемсітушілік: қайталанған шағымдар | 3+ тәуелсіз автордан тіл-кемсіту блогы (отзыв және шағым бірге саналады) |
| Заңсыз айыппұлдар: қайталанған шағымдар | 3+ тәуелсіз автордың отзывында айыппұл-белгісі (ҚР ЕК бойынша жұмыс беруші қызметкерге айыппұл сала алмайды) |

Контент жасырылып сан табалдырықтан түссе, бейдж автоматты қайтарылып алынады. Бұл әділдіктің екі жаққа да жұмыс істейтінінің кепілі.

# Абьюз-қорғаныс (формуладан тыс)

- Бір юзер бір компанияға бір-ақ отзыв қалдыра алады (БД деңгейінде unique constraint).
- Верификациясыз аккаунттың салмағы төмен, жаңа аккаунттарға қосымша шектеулер бар.
- Бейджтердің тәуелсіздік-тексерісі кейін аккаунт жасы мен техникалық сигналдармен күшейтіледі.
